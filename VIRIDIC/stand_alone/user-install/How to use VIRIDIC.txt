# Instructions on how to use VIRIDIC on ODIN
- in case the singularity version of VIRIDIC makes problems (e.g. when passing the blast paramters to VIRIDIC), you can run directly VIRIDIC, without passing through the singularity

 

## How to use VIRIDIC - for other users than cmoraru
* copy to your profile the VIRIDIC scripts found here: /home/cmoraru/software/VIRIDIC/stand_alone/viridic_scripts/

* create and activate the VIRIDIC conda env

	- use the yml file from /home/cmoraru/software/VIRIDIC/stand_alone/user-install/VIRIDIC.yml to create the VIRIDIC conda env
	- install the specific R libraries
	
	
	conda env create -f VIRIDIC.yml
	conda activate VIRIDIC

	# install R libraries from within R
	R  #this command starts R
	options(repos = c(CRAN = "https://cloud.r-project.org/"))

	install.packages("stringr")
	install.packages("magrittr")
	install.packages("dplyr")
	install.packages("tibble")
	install.packages("purrr")
	install.packages("tidyr")
	install.packages("ggplot2")

	install.packages("DT")
	install.packages("shiny")
	install.packages("shinyjs")
	install.packages("shinyWidgets")
	install.packages("seqinr")

				BiocManager::install("IRanges")

*To run VIRIDIC

go to your folder with the VIRIDIC scripts
use command "Rscript 00_viridic_master.R [...options]"
for options, see manual, either by running "Rscript 00_viridic_master.R help" or by checking the PDF file here: /home/cmoraru/software/VIRIDIC/stand_alone/download/viridic_manual_v1.1.pdf
 
# example of how to run one VIRIDIC analysis
	Rscript 00_viridic_master.R projdir=/home/cmoraru/TEST_CLM/testVIridic in=/home/cmoraru/TEST_CLM/Ebeline_rel.fna cpu=10
